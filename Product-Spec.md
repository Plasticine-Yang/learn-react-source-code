# Product Spec：React 源码学习 AI 导师

## 产品概述

一个基于 Tauri v2 的桌面应用，作为 AI 导师帮助开发者循序渐进、深入浅出地学习 React 18.2.0 源码的底层原理。用户通过结构化的课程路径学习，随时与 AI 对话提问，一键跳转到对应源码验证理解，形成"学习 → 提问 → 验证"的闭环。

**目标用户**：有 React 使用经验、想深入理解框架内部机制的初中级前端开发者。他们能写 React 组件但说不清 Fiber 怎么调度、Hooks 怎么存状态、Diff 算法到底做了什么——既想搞懂原理，又觉得源码啃不动。

**核心价值**：不是扔给你一份源码自己读，而是像有个坐在旁边的导师，在你读每一段源码时帮你解释"为什么这么写"、"上下文是什么"、"和之前学的有什么关联"。

## 应用场景

- **系统性学习**：小张有 2 年 React 经验但没读过源码，每天花 30 分钟按课程模块推进，AI 在他看不懂的地方主动解释，学习进度自动保存。

- **面试前突击**：小李下周面试，打开模块 3（Reconciliation）和模块 4（Hooks），让 AI 出几道中等难度的测验题，做完后 AI 指出他对 useEffect 清理时机理解有误，翻出对应源码验证。

- **工作中遇到疑惑**：小陈在项目中用 useMemo 做优化，但不确定是否真的避免了重渲染。他打开 AI 对话面板直接问，AI 引用了 ReactFiberHooks.js 中的相关逻辑，并一键跳转到源码浏览器高亮对应行。

- **源码阅读辅助**：小王在 GitHub 上看 React 源码，Fiber 树的遍历逻辑看不懂。他在 App 内搜索 `workLoopSync`，AI 不仅展示了代码，还用 Mermaid 生成了工作循环的流程图。

- **动手实验**：小刘看完模块 2（Fiber 架构），在沙盒里写了一个会导致大量重新渲染的组件，点击"注入 Fiber 日志"看到实际的遍历过程，直观理解了 Fiber 如何分片工作。

## 功能需求

### 核心功能

- **结构化课程导航**：用户打开应用 → 左侧边栏展示 7 个模块的课程树 → 点击模块进入对应讲义 → 系统渲染 Markdown 内容，包含 AI 生成的引导讲解、关键源码片段（从内置 React 18.2.0 源码提取并语法高亮）、内嵌的 Mermaid 架构图。

- **AI 导师对话**：用户点击右侧对话面板或某段源码旁的"请 AI 讲解"按钮 → 在输入框中输入问题 → AI 以流式输出方式回复，支持 Markdown 和代码块渲染 → AI 可在回复中引用源码文件路径和行号，用户点击引用即可跳转到源码浏览器。

- **源码浏览器**：用户点击左侧"源码"标签或在对话中点击引用 → 左侧显示 React 源码目录树，右侧显示代码内容（Shiki 语法高亮）→ 支持文件名搜索 → AI 引用的行会自动高亮。

- **Agent 工具系统**：AI 对话由 Claude Agent SDK 驱动，具备以下工具：
  - `search_react_source`：全文搜索内置 React 18.2.0 源码
  - `get_source_file`：读取指定文件的特定行范围
  - `explain_with_diagram`：生成 Mermaid 图表解释概念
  - `quiz_user`：根据当前模块出题并评估回答
  - `update_progress`：更新学习进度

- **学习进度追踪**：用户完成模块阅读或通过测验 → 系统自动更新进度百分比 → 左侧课程树显示完成/进行中图标 → 顶部工具栏显示总体进度条 → 学习时长自动记录。

- **实验沙盒**：用户打开沙盒标签 → 左侧是一个简易代码编辑器（Monaco Editor），可写 React 组件 → 右侧是 iframe 渲染的预览 → 提供"注入 Fiber 日志"按钮，在模拟控制台中展示组件渲染时的 Fiber 遍历过程。

- **API 配置**：用户首次打开应用 → 弹出设置向导 → 填写 ANTHROPIC_AUTH_TOKEN（必填）、ANTHROPIC_BASE_URL（可选，默认 api.anthropic.com）、ANTHROPIC_MODEL（可选，默认 claude-sonnet-4-6）→ 密钥通过 Tauri 命令存入系统安全存储，前端不持久化。

### 辅助功能

- **深色/浅色主题切换**：顶部工具栏切换按钮，默认跟随系统主题。
- **侧边栏宽度拖拽**：左侧边栏宽度可通过拖拽调整。
- **对话面板折叠**：右侧对话面板可折叠，专注阅读时隐藏。
- **对话历史管理**：对话自动按模块分组保存到 SQLite，支持查看历史对话、删除单条或清空。
- **测验与成就**：完成模块测验获得徽章，在课程树中显示。
- **学习数据导出**：在设置页导出学习记录（JSON 格式）。

## UI 布局

### 整体布局
三栏布局，左侧边栏（默认 260px，可拖拽）、中央内容区（自适应）、右侧对话面板（默认 360px，可折叠）。

### 顶部工具栏（高 48px，固定）
- 左侧：应用名称 + 当前模块标题
- 中间：模块快速切换下拉框
- 右侧：全局搜索（Ctrl+K）、主题切换按钮、设置齿轮按钮

### 左侧边栏（260px，可拖拽 ±80px）
- 顶部：用户头像 + "学习进度 35%" 进度条
- 中部：课程目录树（7 个模块，展开显示小节）
  - 每个模块前有状态图标：○ 未开始、◐ 进行中、✓ 已完成
  - 已完成模块旁显示徽章图标
- 底部标签切换：「课程」/「源码」/「沙盒」
  - 「源码」：显示 React 源码目录树
  - 「沙盒」：显示沙盒文件列表

### 中央内容区
- 顶部：模块标题 + 学习时长显示
- 主体：Markdown 渲染的课程讲义
  - 代码块使用 Shiki 高亮，右上角显示文件名 + 行号
  - 内嵌 Mermaid 图表
  - 源码片段下方有「请 AI 讲解此段源码」按钮
- 底部：模块导航按钮（上一节 / 下一节）+ 「做个小测」按钮

### 右侧对话面板（360px，可折叠）
- 顶部：面板标题「AI 导师」+ 折叠按钮 + 新建对话按钮
- 主体：对话气泡列表
  - 用户消息靠右，AI 消息靠左
  - AI 消息支持 Markdown 渲染 + 代码块 + Mermaid 图表
  - AI 思考时显示加载动画，流式输出文字
  - AI 引用的源码文件显示为可点击链接
- 底部：输入区域
  - 多行输入框（最少 1 行，最多 6 行自动扩展）
  - Enter 发送，Shift+Enter 换行
  - 发送按钮（输入为空时禁用）

### 设置弹窗（模态框）
- API 配置区：ANTHROPIC_BASE_URL、ANTHROPIC_AUTH_TOKEN、ANTHROPIC_MODEL 三个输入框
- 源码路径：显示「内置 React 18.2.0 源码」
- 学习数据管理：导出记录 / 重置进度
- 关于：版本号、许可信息

## 用户使用流程

### 首次使用
1. 启动应用 → 看到欢迎页 + 设置向导
2. 填写 API 配置（至少填 ANTHROPIC_AUTH_TOKEN）
3. 选择「开始学习」→ 进入模块 1
4. 阅读讲义，点击源码片段旁的按钮请 AI 讲解
5. 完成模块 1 全部小节 → 点击「做个小测」→ AI 出题
6. 通过测验 → 获得徽章 → 自动进入模块 2

### 碎片化学习
1. 打开应用 → 自动恢复到上次阅读位置
2. 在对话面板继续上次的讨论
3. 切换到源码浏览器查某个函数
4. 关闭应用，进度自动保存

### 自由探索
1. 点击左侧「源码」标签 → 展开目录树浏览
2. 搜索 `useState` → 找到 ReactFiberHooks.js
3. 右键选择「请 AI 解释这个函数」→ 对话面板打开并自动提问

## 技术方向

| 维度 | 选择 | 理由 |
|------|------|------|
| 产品类型 | Desktop（Tauri v2） | 需要离线访问本地源码文件、系统级安全存储 API Key、文件系统操作权限 |
| 前端框架 | React 18 + TypeScript + Vite | 与学习目标一致（用户学 React 源码，工具本身用 React 构建）；Vite 开发体验快 |
| 样式方案 | Tailwind CSS | 快速构建 UI，支持暗色模式，无运行时开销 |
| 状态管理 | Zustand | 轻量、TypeScript 友好、适合中等复杂度的客户端状态 |
| AI 引擎 | Claude Agent SDK（@anthropic-ai/claude-agent-sdk） | 内置 Agent 循环、工具调用协议、流式输出，减少手动编排代码 |
| 代码高亮 | Shiki | 服务端/构建时高亮，精度高，支持 React 源码中使用的 JS/TS/Flow |
| 内容渲染 | react-markdown + remark-gfm + Mermaid | Markdown 标准方案，GFM 支持表格/任务列表，Mermaid 渲染架构图 |
| 沙盒编辑器 | Monaco Editor | 完整 IDE 体验，JSX 语法支持好 |
| 数据持久化 | SQLite（tauri-plugin-sql） | 对话历史、学习进度结构化管理，本地存储不需要服务器 |
| 安全存储 | tauri-plugin-store | Tauri 官方方案，加密存储 API Key，前端不可直接访问 |
| 部署方式 | Tauri 桌面安装包（.deb / .AppImage / .dmg / .msi） | 桌面应用标准分发方式 |

## 技术说明

- **React 源码内置**：React 18.2.0 完整源码以资源形式打包在 Tauri 中（约 30MB），应用首次启动时解压到本地数据目录，供源码浏览器和 Agent 搜索工具使用。
- **AI 调用架构**：前端通过 Tauri `invoke` 命令将对话请求发送到 Rust 后端，Rust 端从安全存储读取 API Key 后转发到 Anthropic API。Agent 循环在前端实现，工具调用结果由 Rust 命令返回。
- **流式响应**：Rust 端通过 `tauri::ipc::Channel` 将 SSE 流逐块转发到前端，确保打字机效果不卡顿。
- **离线行为**：无网络时，课程内容、源码浏览、沙盒可正常使用；AI 对话和搜索功能不可用，显示离线提示。
- **API Key 配置**：支持三个环境变量风格的配置项：`ANTHROPIC_BASE_URL`（默认 `https://api.anthropic.com`）、`ANTHROPIC_AUTH_TOKEN`（必填，用户的 API Key）、`ANTHROPIC_MODEL`（默认 `claude-sonnet-4-6`），兼容自定义 API 代理。
