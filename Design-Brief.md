# Design Brief：React 源码学习 AI 导师

## 产品概要
- **产品名称**：React 源码学习 AI 导师
- **产品类型**：Desktop（Tauri v2）
- **目标用户**：有 React 使用经验、想深入理解框架内部机制的初中级前端开发者
- **核心功能**：结构化课程学习（7 模块 31 节）、AI 导师对话（Claude Agent SDK + 工具调用）、源码浏览器（React 18.2.0 内置）、实验沙盒（Monaco + iframe 预览）、学习进度与成就追踪

## 设计方向

### 情绪关键词
- **智能** — AI 驱动的对话导师，主动解释源码、出题检验、生成图表；视觉上通过流式输出和打字效果传递"思考和响应"感
- **极简** — 每像素都有原因。UI chrome 刻意压低，信息层级用分隔线而非卡片区分，内容区视觉权重最高
- **未来感** — Tokyo Night 冷暗色调营造沉浸式夜间城市氛围，蓝紫霓虹点缀，代码区是第一视觉主角

### 参考产品

| 参考产品 | 喜欢的方面 | 不喜欢的方面 |
|---------|-----------|-------------|
| VS Code | 源码浏览区原生编辑器感——文件树 + 标签页 + 行号 + 语法高亮，信息密度高效 | UI chrome 有些区域偏紧凑拥挤 |
| Cursor | AI 对话面板与代码区的整合方式，Agent 交互思维 | — |
| Claude Code | 极简排版——等宽字体主导、分隔线划区、流式逐行呈现、几乎没有 UI chrome | — |
| Tokyo Night Theme | 全局色彩系统——深蓝紫底、低饱和度 UI、冷色霓虹语法高亮、刻意压低的侧边栏对比度 | — |

### 反面参考
- 用户未指出明确排斥的风格

## 视觉规范

### 色彩方向
- **主题模式**：深色默认（Tokyo Night），也做浅色（Tokyo Night Day），自动跟随系统但可手动切换
- **色彩温度**：冷色系——蓝紫调为主
- **品牌主色方向**：完整使用 Tokyo Night 调色盘，不额外添加品牌色。深色背景基准 `#1a1b26`（Night）/ `#24283b`（Storm），主文本 `#c0caf5`（冰蓝白），UI 辅助文本压低对比度
- **强调色方向**：函数名/交互元素用 `#7aa2f7`（矢车菊蓝），关键字/控制流用 `#bb9af7`（柔紫），错误/警告用 `#f7768e`（粉红），成功状态用 `#9ece6a`（柔绿）
- **品牌资产**：无，从零构建

### 信息密度
- **密度方向**：适中留白——三栏同时可见，但内容区有呼吸空间
- **参考基准**：比 VS Code 多一层间距，比 Notion 紧凑；介于 IDE 与文档阅读器之间
- **理由**：三栏布局信息量大（课程树 + 讲义 + 对话），但核心任务是阅读学习，不能挤。代码块撑满宽度，文字区左右留适当间距

### 排版方向
- **字体气质**：机械无衬线正文 + 等宽代码，干净利落，不加修饰
- **等宽字体**：JetBrains Mono 或 Fira Code，用于代码块、源码浏览器、对话中的代码片段
- **标题风格**：层级清晰但不过度夸张，用字号和间距区分而非颜色和装饰线
- **正文排版**：Claude Code 风格——信息块之间用细分隔线（非卡片），内容流式呈现，段落间距宽松

### 交互风格
- **动画程度**：细腻但不慢——面板展开/折叠用 ease-out 缓动（~200ms），模块切换淡入淡出，AI 回复流式打字机效果，hover 微反馈
- **过渡效果**：fast ease-out，不弹跳、不弹簧、不减速拖沓
- **整体节奏**：沉稳，不抢注意力，让用户聚焦内容

## 核心页面视觉备注

### 课程讲义区（中央内容区）
- **核心交互**：阅读 Markdown 渲染的课程内容，包含代码块、Mermaid 图表、源码引用链接
- **视觉方向**：技术文档风格——像 React 官方文档 / Rust Book 的排版，层级清晰、代码块突出（Shiki 高亮 + 文件名 + 行号）、标题锚点导航。读代码是主任务
- **特殊要求**：源码片段下方有「请 AI 讲解此段源码」按钮（`#7aa2f7` 轮廓样式）；底部有「做个小测」入口按钮；模块底部有上一节/下一节导航

### AI 对话面板（右侧，可折叠）
- **核心交互**：用户输入消息，AI 流式回复（Markdown + 代码块 + Mermaid），AI 可自主调用工具并展示调用状态
- **视觉方向**：软气泡——AI 消息气泡用 Tokyo Night Storm 色 `#24283b` 背景，用户消息气泡用 `#7aa2f7` 半透明背景。圆角卡片，左右对齐区分角色。AI 思考时显示 typing indicator（三点脉动，`#7aa2f7`）
- **特殊要求**：AI 引用的源码文件显示为可点击链接（`#7dcfff` 下划线样式），点击跳转源码浏览器对应行；工具调用展示可折叠的状态卡片

### 源码浏览器（左侧标签页切换）
- **核心交互**：左侧 React 源码目录树 + 右侧代码展示（语法高亮 + 行号 + 标签页）
- **视觉方向**：VS Code 原生编辑器的感觉——文件树 + 标签页 + 行号 + Shiki 语法高亮。代码区背景稍亮于 UI 背景（`#1a1b26` → 代码区 `#1e2030`）
- **特殊要求**：AI 引用的行高亮（`#7aa2f7` 10% 透明背景 + 左侧竖线标记）；支持文件名搜索（Cmd/Ctrl+F）

### 侧边栏导航（左侧）
- **核心交互**：课程目录树 + 底部源码/沙盒标签切换
- **视觉方向**：Tokyo Night 低对比度 UI chrome——背景不抢眼，当前模块用 `#7aa2f7` 文字高亮 + 左竖线指示，未完成/进行中/已完成用图标区分
- **特殊要求**：宽度可拖拽（默认 260px，范围 180-340px），底部有总体进度条

### 实验沙盒
- **核心交互**：Monaco 编辑器（左）+ iframe 预览（右），嵌入在课程讲义内作为可交互示例
- **视觉方向**：嵌入式——沙盒区域有浅边框与课程内容区隔，编辑器部分缩小版 VS Code 感觉，"注入 Fiber 日志"按钮触发 log 输出面板
- **特殊要求**：不作为独立页面，而是课程内容中的嵌入式区域，有明确的"这是一个可运行的例子"视觉提示

### 设置页
- **核心交互**：模态弹窗，包含 API 配置、学习数据管理、关于信息
- **视觉方向**：表单式布局——输入框用 Tokyo Night UI 底色 + `#7aa2f7` 焦点边框，分组用细分隔线，保存按钮用 `#7aa2f7` 实色
- **特殊要求**：ANTHROPIC_AUTH_TOKEN 输入框为密码类型，显示/隐藏切换

## 状态设计
- **空状态**：轻量几何插画（Tokyo Night 色系单色线条风，如一个空的文件图标、或一个休眠的聊天泡泡）+ 一行引导文案 + 一个操作按钮（`#7aa2f7` 轮廓样式）
- **加载状态**：AI 回复用打字机光标动画（竖线闪烁），内容面板加载用骨架屏（底色 `#24283b` + shimmer 动画），模块切换用弱透明遮罩 + 微旋转指示器（仅长加载时出现）
- **错误状态**：内联错误提示，用 `#f7768e` 做左边框强调 + 浅色粉红背景，不用弹窗打断操作。网络错误在对话面板底部显示可重试按钮
